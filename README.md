# Aletheia — Unsafe Transaction Types for Aiken

An Aiken library that provides unwrapped transaction types that survive serialization through Plutus Data, plus conversion functions into the stdlib.

## What it does

Aiken's `cardano/transaction.Transaction` is opaque. It enforces internal invariants (`Value` canonical ordering, uniqueness guarantees, etc.) that make it impossible to round-trip through `serialise` / `deserialise`. If you try to `deserialise` a stdlib `Transaction` from Plutus Data, the compiler rejects it.

This is a problem when you want to test validators against **real transaction data** from the blockchain. You can't just serialize a transaction off-chain, paste it into Aiken, and `deserialise` it.

Aletheia solves this by providing **unsafe** types built from only simple structures — plain records, lists, pairs, and primitives — that carry the same data as their stdlib counterparts but without the hidden invariants. These types:

- Compile to the same Plutus Data shape as the off-chain TypeScript schemas used by [Hermes](https://github.com/txpipe-shop/hermes).
- Can be `deserialise`d from CBOR generated by Hermes (or any tool that outputs matching Plutus Data).
- Can be converted into the standard stdlib `Transaction` via the `convert` module, so your validators work with the familiar types.

In short: Aletheia is the on-chain half of the Hermes workflow. Hermes exports CBOR; Aletheia deserializes and converts it.

```
┌────────────────────────────────────────────────────────────────┐
│  TypeScript (off-chain)                Aiken (on-chain)        │
│                                                                │
│  Raw TX CBOR                       deserialise CBOR            │
│       │                                  │                     │
│       ▼                                  ▼                     │
│  ┌──────────────┐                 ┌──────────────────┐         │
│  │ Hermes       │                 │ UnsafeTransaction│         │
│  │              │  serialize ───▶ │ (aletheia)       │         │
│  │ UnsafeTx obj │   to CBOR       │                  │         │
│  └──────────────┘                 └────────┬─────────┘         │
│                                            │                   │
│                                            │ convert.ak        │
│                                            ▼                   │
│                                  ┌──────────────────┐          │
│                                  │ Transaction      │          │
│                                  │ (stdlib)         │          │
│                                  └──────────────────┘          │
└────────────────────────────────────────────────────────────────┘
```

## Installation

Add aletheia as a dependency in your `aiken.toml`:

```toml
[[dependencies]]
name = "txpipe-shop/aletheia"
version = "0.0.0"
source = "github"
```

Then import it in your Aiken code:

```aiken
use aletheia/unsafe_transaction.{
  UnsafeTransaction
}
use aletheia/convert.{
  unsafe_transaction_to_transaction
}
```

## Usage

Aletheia is designed to be used alongside test code generated by [Hermes](https://github.com/txpipe-shop/hermes). Hermes emits `.ak` files that contain the serialized transaction CBOR, redeemer data constants, and policy hashes — everything needed to reconstruct the script context.

The Hermes-generated code handles deserialization and conversion to a proper `Transaction`. You write the test logic and validator calls on top of it.

### Generated by Hermes

```aiken
// ── Hermes-generated preamble ──────────────────────────────────
use aletheia/unsafe_transaction.{UnsafeTransaction}
use aletheia/convert.{unsafe_transaction_to_transaction}

const mint_1_redeemer: Data = {
  expect Some(d) = deserialise(#"d87980")
  expect r: Data = d
  r
}
const mint_1_policy = #"b6794b9ab91aa3d8ab704623c38117f54b4d5c082d9d36a09fea4400"

const tx_ut: Transaction = {
  expect Some(unsafe_tx_data) = deserialise(
    #"d8799f9fd8799f5820...ffff",
  )
  expect unsafe_tx: UnsafeTransaction = unsafe_tx_data
  unsafe_transaction_to_transaction(unsafe_tx)
}
// ── End Hermes-generated ────────────────────────────────────────
```

### Written by you

```aiken
// ── Your test logic ─────────────────────────────────────────────
test mint_1_validation() {
  // tx_ut, mint_1_redeemer, mint_1_policy are all generated above
  my_script.mint(mint_1_redeemer, mint_1_policy, tx_ut)
}
```

> **Note:** The `tx_ut` block deserializes the CBOR into an `UnsafeTransaction`, then converts it to a stdlib `Transaction` in one step. You get a fully populated `Transaction` ready to pass directly to your validator's purpose handlers.
